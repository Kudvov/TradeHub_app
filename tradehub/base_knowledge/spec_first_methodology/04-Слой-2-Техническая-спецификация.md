# 04. Слой 2 - Техническая спецификация

Backlinks: [[README]] [[03-Слой-1-Документ-идеи]]

## Цель
Трансформировать идею в формат, готовый для реализации без уточнений.

## 6 блоков каждого модуля
1. User Stories.
2. Модель данных (типы SQL, constraints, FK, RLS).
3. API и серверная логика.
4. Экраны и компоненты.
5. Бизнес-логика.
6. Крайние случаи.

## Формат важнее красоты
- Типы как в SQL (`uuid`, `text`, `jsonb`, `timestamptz`).
- API как в коде (`POST /api/...`, тела, ответы, коды ошибок).

## Правило
Если ответ уже есть в исходных документах, AI не должен задавать уточняющие вопросы.
