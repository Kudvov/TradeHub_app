## 2026-04-03
- **Multilingual site (RU / EN / KA)** — full i18n implementation:
  - `svelte-i18n` library; locales in `src/lib/locales/ru.json`, `en.json`, `ka.json` (~80 keys).
  - `LanguageSwitcher.svelte` added to Header.
  - Locale saved to `localStorage`, falls back to browser navigator locale.
  - All UI strings translated: categories, cities, filters, dates, errors, buttons, report modal.
  - `formatDate` and `formatPrice` made locale-aware.
- **Listing translations (Claude Haiku API)**:
  - New DB columns: `title_en`, `description_en`, `title_ka`, `description_ka`.
  - Migration: `scripts/migrations/2026-04-listings-translations.sql`.
  - Script: `src/lib/server/telegram/translate-listings.ts` — batches of 20, `max_tokens: 2048`, Claude Haiku (`claude-haiku-4-5`).
  - Run: `npm run parser:translate` — incremental, skips already-translated.
- **Listing detail — translation toggle**:
  - Default: show translation if available.
  - Toggle buttons «Перевод» / «Оригинал» — only shown when `hasTranslation = true`.
  - Resets to translation view on locale change.
- **ListingCard — language badge**:
  - RU flag badge shown on cards when locale is EN or KA.
  - Dimmed if listing not yet translated.
- **AI reclassification** (`parser:ai-reclassify`):
  - All DB listings re-classified via Ollama (qwen2.5:3b).
  - Ollama CPU limited to 3 cores via systemd `CPUQuota=300%`.
