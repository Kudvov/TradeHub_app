# Data Model

Backlinks: [[README]] [[02-Architecture/System-Architecture]] [[03-Backend/Parser-Pipeline]] [[06-Incidents/Deduplication]]

## Core tables
- `cities`: city metadata + listings counters.
- `telegram_groups`: source groups, status, cursors (`lastMessageId`, `startMessageId`).
- `categories`: normalized category taxonomy.
- `listings`: main listing entity with Telegram linkage, `images` array, and multilingual translations.
- `listing_reports`: moderation reports from UI.
- `banned_authors`: blocked contacts.

## Important fields in `listings`
- `telegramMessageId`: source post ID in Telegram group.
- `telegramGroupId`: source group relation.
- `contentHash`: dedupe key within a city (`cityId` + hash for active rows); see [[06-Incidents/Deduplication]].
- `images`: array of image URLs (can be refreshed).
- `status`: `active` or filtered states.
- `publishedAt`: ranking/freshness anchor.
- `checkedAt`: last time listing was validated on t.me.

## Translation fields in `listings`
Added 2026-04 via `scripts/migrations/2026-04-listings-translations.sql`:

| Column | Type | Description |
|--------|------|-------------|
| `title_en` | text | English title (Claude Haiku) |
| `description_en` | text | English description |
| `title_ka` | text | Georgian title |
| `description_ka` | text | Georgian description |

- Source language is always Russian (original Telegram posts).
- Translations are generated by `parser:translate` using Claude Haiku API (`claude-haiku-4-5`).
- NULL = not yet translated. Frontend falls back to Russian original when NULL.
- Batched 20 listings at a time, `max_tokens: 2048`.
- Script: `src/lib/server/telegram/translate-listings.ts`

## Data integrity rules
- Keep `telegramMessageId` immutable for identity.
- Only update mutable content (`images`, category/status corrections, translations) when reparsing.
- Recompute city counters after parser/reparse operations.
- Never overwrite existing non-null translations without an explicit re-translate run.
