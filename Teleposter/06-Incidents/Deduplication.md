# Дубликаты объявлений (между группами)

Backlinks: [[README]] [[03-Backend/Parser-Pipeline]] [[02-Architecture/Data-Model]]

## Проблема
Один и тот же текст объявления мог приходить из разных Telegram-групп в одном городе. Раньше дедупликация опиралась в основном на пару группа + сообщение, из-за чего в проде могли появляться дубли с одинаковым `contentHash`.

## Текущее поведение (sync)
- **Идентичность поста в группе:** проверка по `telegramGroupId` + `telegramMessageId` (одно и то же сообщение в той же группе).
- **Межгрупповой дедуп в городе:** если для активного объявления в том же `cityId` уже есть запись с тем же `contentHash`, новый пост не вставляется как отдельное объявление; при необходимости объединяются изображения с существующей записью.

## Очистка старых дублей в БД
Разовый или периодический скрипт: `npm run parser:cleanup-duplicates`.

| Режим | Как включить |
|--------|----------------|
| Dry-run (только отчёт) | по умолчанию |
| Применить (`status = filtered` для дублей, keeper обновляется) | `APPLY=1` |

Опционально: `DUPLICATE_GROUPS_LIMIT` — ограничить число групп дублей за прогон.

Keeper — запись с наиболее свежим `publishedAt` (при равенстве — больший `id`); в неё сливаются картинки и недостающие поля; остальные активные копии помечаются `filtered`.

## Связанный код
- `tradehub/src/lib/server/telegram/sync.ts` — логика при импорте.
- `tradehub/src/lib/server/telegram/cleanup-duplicates.ts` — офлайн-очистка.
