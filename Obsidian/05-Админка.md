# Админка

## URL

- **`/admin/groups`** — вход по паролю: cookie `teleposter_admin_session` выставляется после POST с `ADMIN_KEY` из `.env` (см. `+page.server.ts`).

## Возможности

- Вкладки: парсер (статус systemd, счётчики новых объявлений за 5/15/60 мин), логи journalctl, группы, аналитика, жалобы, справка.
- Действия парсера: запуск unit, старт/стоп/enable/disable таймера — **только если процесс Node на сервере может выполнять `systemctl`** (обычно root или политики sudo).

## API

- `GET /api/admin/live-status` — счётчики + `systemctl is-active teleposter-parser.service`
- `GET /api/admin/parser-logs` — текст лога для вкладки «Логи»

Ответы с заголовками без кэша, чтобы «Обновить лог» показывал актуальные строки.
